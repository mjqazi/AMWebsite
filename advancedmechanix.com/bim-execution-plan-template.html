BIM Execution Plan template — free, ISO 19650-aligned.

An editable BEP template covering everything a BIM project needs to define before modeling starts: LOD requirements, model federation, file naming, clash workflow, and deliverable schedule. Free download.

§01 — What's in the template

Twelve sections every BEP needs.

Most BIM projects fail not because the modeling is wrong but because the BEP was rushed, vague, or skipped entirely. This template is the minimum scaffolding for a usable BEP — structured around the workflow we use internally on every BIM engagement.

01

Project Overview & Stakeholders

Project identification, design team, client BIM POC, BIM manager, deliverable approver chain.

02

BIM Goals & Use Cases

What the model is for: design coordination, clash detection, fabrication, facility management, regulatory submission.

03

LOD Requirements

LOD per discipline per project phase. LOD 350 explained.

04

Model Federation Strategy

Discipline split, federation point, central-model architecture, worksets.

05

Coordinate System & Origin

Project base point, survey point, shared coordinates, true north vs project north.

06

File Naming Convention

Naming standard for model files, drawings, deliverables. ISO 19650-aligned where applicable.

07

Software & Versions

Specific software versions per discipline, file format compatibility, plugin requirements.

08

Content & Family Library

Source of approved Revit families, custom family workflow, parameter standards.

09

Clash Detection Workflow

Frequency, tooling (Navisworks / BIM Collab), severity classification, resolution responsibility.

10

Quality Control Process

Internal QA gates, peer review, audit checkpoints, deliverable sign-off process.

11

Deliverable Schedule

Milestone deliverables, format, recipient, approval cycle, revision threshold.

12

Common Data Environment

CDE platform (BIM 360 / ProjectWise / Aconex), folder structure, access permissions, version control.
